# ISO/IEC 27002 Guidance (Demo)

## Access Control
- Provision access based on least privilege and role-based profiles.
- Review privileged accounts every quarter; enforce MFA for admin login.

## Backup
- Align backup schedules with data classification: critical systems daily, others weekly.
- Test restorations quarterly and log results in the change register.

## Cryptographic Controls
- Maintain key management covering generation, storage, rotation, and revocation.
- Use vetted algorithms (AES-256, RSA-2048) and document key custody.

## Supplier Security
- Conduct security due diligence before onboarding vendors; require contractual clauses.
- Monitor supplier performance with annual reviews and security questionnaires.

## Mobile & Remote Working
- Enforce device encryption, screen-lock timers, and MDM policies.
- Require VPN or zero-trust access from remote networks.

## Incident Response & Change Management
- Maintain runbooks with detection, triage, communication, and lessons learned.
- Ensure change requests include security impact analysis and rollback plans.
