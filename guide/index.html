Step 1 – Prepare Inputs

  • Create .env with keys:
    HUGGINGFACE_EMBEDDING_MODEL=BAAI/bge-m3
    Optional: OPENAI_API_KEY, OPENAI_BASE_URL, OPENAI_MODEL_NAME.
  • Edit memory/system_context.json to reflect current assets, risks, policies, treatment plans.
  • Update knowledge_base/*.md with fresh ISO guidance or internal docs (copy text from PDFs into Markdown).

Step 2 – Install Locally

Run these once per machine:

git clone <repo-url>
cd iso27001
uv sync
# macOS Intel reset (optional):
rm -rf .venv uv.lock
uv venv --python 3.11
source .venv/bin/activate
uv sync

Step 3 – Build the RAG Database

Re-run any time you change files in knowledge_base/.

uv run python -m iso27001.knowledge.rag_loader

Check vector_db/iso27001_knowledge for confirmation (or the container volume when using Docker).

Step 4 – Execute the Crew

Local:

uv run crewai run

Docker:

docker build -t iso27001-crew .
# optional: rebuild embeddings inside container
 docker run --rm \ 
   -v $(pwd)/vector_db:/app/vector_db \ 
   --env-file .env \ 
   iso27001-crew uv run python -m iso27001.knowledge.rag_loader
# main run
 docker run --rm \ 
   -v $(pwd)/outputs:/app/outputs \ 
   -v $(pwd)/vector_db:/app/vector_db \ 
   --env-file .env \ 
   iso27001-crew

Step 5 – Review Outputs

  • Open outputs/asset_register.md to confirm the asset inventory.
  • Check outputs/risk_register.md and outputs/risk_treatment_plan.md.
  • Download policies from outputs/policies/security_policies.md.
  • Inspect outputs/statement_of_applicability.md for Annex A coverage.
  • Share outputs/iso27001_readiness_report.md with leadership/auditors.
Need to rerun? Update memory or knowledge base, rebuild embeddings, and run the crew again. All steps live in this single page for ease of use.
